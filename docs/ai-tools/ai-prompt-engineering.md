# AI Prompt 工程完全指南

Prompt（提示词）是与 AI 交互的核心。写好 Prompt 决定了 AI 输出内容的质量和准确度。

## 什么是 Prompt 工程

Prompt 工程是指设计和优化输入给 AI 的文本指令，以获得更准确、更有用的输出结果。它不是编程，而是一种"用自然语言编程"的技能。

## Prompt 的基本结构

一个高质量的 Prompt 通常包含以下要素：

1. **角色设定**：告诉 AI 以什么身份回答
2. **任务描述**：明确要做什么
3. **上下文信息**：提供必要的背景信息
4. **输出格式**：指定结果的呈现方式
5. **约束条件**：限制范围、风格、长度等

## 实用 Prompt 模板

### 内容创作

```
你是一位资深科技编辑，擅长撰写通俗易懂的技术文章。
请围绕"AI 如何改变办公方式"写一篇 1500 字的文章。
要求：
- 使用 3-5 个具体案例
- 包含数据支撑的观点
- 语言风格轻松但不失专业
- 文末给出 3 条行动建议
```

### 代码辅助

```
你是一个有 10 年经验的 Python 高级工程师。
请审查以下代码，指出：
1. 存在的潜在 Bug
2. 性能优化建议
3. 代码风格改进
4. 安全性问题

代码：
```python
# 粘贴你的代码
```
```

### 数据分析

```
假设你是一位数据分析师。我有以下销售数据：
[粘贴数据]

请帮我：
1. 总结关键趋势（3-5 条）
2. 找出异常数据点
3. 给出下季度的预测方向
4. 用表格形式呈现分析结果
```

## 高级技巧

### 思维链（Chain of Thought）

让 AI 逐步推理，而不是直接给出答案：

```
请一步步分析这个问题，展示你的推理过程，最后再给出结论。
```

### 少样本学习（Few-Shot）

提供几个示例让 AI 学习模式：

```
将以下口语化表达转换为正式商务语言：

示例 1：
输入：这个东西太贵了，能不能便宜点
输出：该产品的定价高于市场平均水平，请问是否有价格优惠的空间？

示例 2：
输入：我觉得这个方案不行
输出：我认为该方案在可行性方面存在一定挑战

请转换：我想快点把这个搞完
```

### 角色扮演

```
你现在是一位面试官，正在面试一位有 3 年经验的前端工程师。
请列出 10 个由浅入深的面试问题，涵盖 JavaScript、React 和系统设计。
对每个问题标注考察点和期望的回答深度。
```

## 常见错误

| 错误 | 改进方式 |
|------|---------|
| 太笼统 | 增加具体约束和格式要求 |
| 一次问太多 | 拆分为多个独立 Prompt |
| 缺少上下文 | 补充背景信息和参考资料 |
| 没有示例 | 加入输入输出示例 |
| 忽略迭代 | 根据输出结果不断调整 Prompt |

## 不同 AI 的 Prompt 差异

- **ChatGPT**：偏好结构化 Prompt，支持系统角色设定
- **Claude**：擅长长上下文理解，适合复杂分析任务
- **Gemini**：与 Google 生态整合，适合搜索增强任务
- **通义千问**：中文理解能力强，适合中文内容创作

## 总结

Prompt 工程是一门"学起来简单、精通难"的技能。核心原则是：**具体明确、提供上下文、迭代优化**。随着实践的积累，你会逐渐形成自己的 Prompt 模板库，大幅提升 AI 使用效率。
